# Dragon Roadmap

This is intentionally product-level. Do not start implementation from this file
alone.

## Phase 0: Reference And Content Bible

- Inventory classic data shapes.
- Maintain Dragon's own committed `.dat` tables under `assets/dragon/dats/`.
- Decide Dragon setting, tone, and core NPC cast.
- Create original monster level buckets.
- Create original weapon and armor tier tables.
- Define the first event schema.
- Draft the first Daily Happenings template set.
- Define moderation boundaries for user-authored text.

## Phase 1: Playable Daily Core

- Character creation.
- Town square.
- Status view.
- Forest fight.
- Monster rewards.
- Death/recovery.
- Bank.
- Healer.
- Weapon and armor shops.
- Daily reset.
- Daily happenings log.

## Phase 2: Progression And Rivalry

- Trainer/master fights.
- Level advancement.
- Player rankings.
- PvP attempts.
- Attack result news.
- Player-authored taunts or last words.
- Gold theft or loot rules.

## Phase 3: Social Gravity

- Mail/notes.
- Flirt/proposal relationship actions.
- Inn/rest state.
- Gossip and rumor events.
- NPC insults and praise.
- Public embarrassment events.
- Relationship status shown in rankings/profile.

## Phase 4: Depth

- Skill paths.
- Hidden locations.
- Rare event chains.
- Dragon challenge.
- Post-dragon legacy reset.
- Admin tuning tools.
- Event authoring workflow.
